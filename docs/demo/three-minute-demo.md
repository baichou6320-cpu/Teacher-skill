# Teacher-skill 3 分钟固定演示脚本

目标：让大老板在 3 分钟内看懂你独立落地了一个完整 AI 学习产品，而不是只看见一段代码或一个聊天 Demo。

## 演示前准备

```powershell
cd C:\Users\wujiehang\Desktop\Teacher-skill\Teacher-skill
.\.venv\Scripts\python.exe main.py --check
```

如果环境检查全绿，正式演示用固定离线脚本：

```powershell
.\.venv\Scripts\python.exe demo_full_loop.py
```

如果现场时间很紧，或想先快速预演：

```powershell
.\.venv\Scripts\python.exe demo_full_loop.py --fast
```

## 3 分钟话术

### 0:00 - 0:20 开场

“这个项目叫 Teacher-skill。它不是做一个更快的总结器，而是把学习材料变成一个可验证的学习闭环：材料输入、自动拆知识点、讲解、提问、判卷、记录弱点、后续复习。”

### 0:20 - 0:45 输入材料

屏幕会显示一段 Transformer 学习材料。

“我从普通文本开始，不需要提前手工做题库。这个入口未来可以接文章、笔记、PDF。”

### 0:45 - 1:10 自动拆卡片

屏幕会出现 3 张知识卡片。

“系统把长材料拆成最小学习单元。每张卡片都有标题、验证问题、正确答案和提示素材。这里体现的是产品化能力：不是只调用模型，而是把模型输出变成可控的数据结构。”

### 1:10 - 1:45 答对闭环

第一题用户答对，系统给出通过反馈。

“这一步展示讲解和验证。用户不是被动看答案，而是必须主动回答，系统再判断是否真的理解。”

### 1:45 - 2:20 答错和速查

第二题用户答错，系统先给提示，然后用户使用 `/direct`。

“答错时不会马上公布完整答案，而是先给线索。如果用户赶时间，可以用速查，但系统会把这个知识点标记为待巩固，后续复习会优先出现。”

### 2:20 - 2:45 保存恢复和报告

屏幕会显示学习状态：2 个已掌握，1 个待巩固。

“这里展示完整闭环：每个知识点的状态、尝试次数、提示层级都会保存。它不是一次性的聊天记录，而是一个可以恢复和复习的学习系统。”

### 2:45 - 3:00 收束

“我想展示的是：我可以从产品目标出发，把一个 AI 想法落成可运行、可检查、可测试、可演示的项目。这里已经包含 CLI 入口、状态机、模型协议、持久化、测试和固定演示路径。”

## 现场备用方案

如果网络或 API 不稳定，只演示固定脚本，因为它不调用外部 LLM/API。

如果老板想看真实模型调用，再补充运行：

```powershell
.\.venv\Scripts\python.exe main.py --demo
```

但正式 3 分钟展示不要优先用 `main.py --demo`，因为它依赖现场网络和模型响应速度。
